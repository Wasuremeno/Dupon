# Задание 2: Автоматический бэкап WordPress

## Задача
Настроить ежедневное резервное копирование сайта на WordPress (файлы + БД) с отправкой отчета в Telegram.

## Решение

### Основной подход
Bash-скрипт + cron + Telegram Bot API

### Почему bash?
- Минимум зависимостей
- Прозрачность (легко читать и дебажить)
- Работает на любом Linux-сервере
- Простота интеграции с cron

### Структура решения
bash-скрипт → mysqldump (БД) + rsync (файлы) → tar (архив) → Telegram (отчет) → ротация (очистка старых бэкапов)


### Компоненты

| Компонент | Инструмент | Назначение |
|-----------|------------|------------|
| Бэкап БД | mysqldump | Создание дампа базы данных |
| Бэкап файлов | rsync/tar | Копирование и архивация файлов сайта |
| Планировщик | cron | Ежедневный запуск |
| Уведомления | Telegram Bot API | Отправка отчетов |
| Ротация | find + rm | Удаление старых бэкапов |

### Скрипт бэкапа

Основной скрипт: [scripts/backup-wordpress.sh](./scripts/backup-wordpress.sh)

**Что делает скрипт:**
1. Загружает конфигурацию из файла
2. Создает временную директорию
3. Делает дамп базы данных (mysqldump)
4. Копирует файлы сайта (rsync с исключениями)
5. Создает итоговый архив (tar)
6. Отправляет отчет в Telegram
7. Удаляет старые бэкапы (ротация 7 дней)
8. Очищает временные файлы

### Конфигурация

Файл конфигурации: [config/backup-config.env.example](./config/backup-config.env.example)

```bash
# Параметры сайта
SITE_PATH="/var/www/wordpress"
DB_NAME="wordpress_db"
DB_USER="wp_user" 
DB_PASS="secure_password"

# Параметры бэкапа
BACKUP_DIR="/backups/wordpress"
BACKUP_RETENTION_DAYS=7

# Telegram
TELEGRAM_TOKEN="ваш_токен_бота"
CHAT_ID="ваш_chat_id"

# Опционально: уведомления об ошибках
ERROR_CHAT_ID="id_чата_для_ошибок"
